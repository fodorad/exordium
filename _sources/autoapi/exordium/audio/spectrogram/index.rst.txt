.. py:function:: compute_mfcc(y: torch.Tensor, sample_rate: int, n_mfcc: int = 40, n_mels: int = 80, n_fft: int = 512, hop_length: int = 160, log_mels: bool = True, save_fig: bool = False, output_path: pathlib.Path | str | None = None) -> tuple[numpy.ndarray, numpy.ndarray]

   Compute MFCC features from audio signal.

   Defaults are tuned for speech emotion and sentiment analysis at 16 kHz:

   * ``n_mels=80`` — covers all perceptually relevant speech frequencies
     (0–8 kHz) without the redundancy of 128 bands, which are better suited
     to music.  Used by Whisper, WavLM, and most SER benchmarks.
   * ``n_fft=512`` — 32 ms window gives stable frequency estimates while
     preserving temporal resolution for fast prosodic changes.
   * ``hop_length=160`` — 10 ms hop matches the frame rate expected by most
     pretrained speech models and standard SER evaluation protocols.

   The torchaudio default of ``n_fft=400`` with ``n_mels=128`` produces only
   201 frequency bins, leaving the highest mel bands empty and triggering a
   ``UserWarning``.  The chosen ``n_fft=512`` yields 257 bins, which is
   sufficient for 80 mel bands with no empty filterbanks.

   :param y: 1-D mono waveform tensor.
   :param sample_rate: Sample rate in Hz.
   :param n_mfcc: Number of MFCC coefficients. Defaults to 40.
   :param n_mels: Number of mel filterbanks used internally. Defaults to 80.
   :param n_fft: FFT size (window length in samples). Defaults to 512.
   :param hop_length: Hop size between frames in samples. Defaults to 160.
   :param log_mels: Whether to use log mel spectrogram. Defaults to True.
   :param save_fig: Whether to save figures. Defaults to False.
   :param output_path: Directory to save figures.

   :returns: Tuple of ``(mfcc, mfcc_preemph)`` as ``(n_mfcc, T)`` numpy arrays.


.. py:function:: compute_melspec(y: torch.Tensor, sample_rate: int, n_mels: int = 80, n_fft: int = 512, hop_length: int = 160, f_max: int = 8000, save_fig: bool = False, output_path: pathlib.Path | str | None = None) -> tuple[numpy.ndarray, numpy.ndarray]

   Compute Mel spectrogram features from audio signal.

   Defaults are tuned for speech emotion and sentiment analysis at 16 kHz:

   * ``n_mels=80`` — covers all perceptually relevant speech frequencies
     (0–8 kHz) without the redundancy of 128 bands, which are better suited
     to music.  Used by Whisper, WavLM, and most SER benchmarks.
   * ``n_fft=512`` — 32 ms window gives stable frequency estimates while
     preserving temporal resolution for fast prosodic changes.
   * ``hop_length=160`` — 10 ms hop matches the frame rate expected by most
     pretrained speech models and standard SER evaluation protocols.

   The torchaudio default of ``n_fft=400`` with ``n_mels=128`` produces only
   201 frequency bins, leaving the highest mel bands empty and triggering a
   ``UserWarning``.  The chosen ``n_fft=512`` yields 257 bins, which is
   sufficient for 80 mel bands with no empty filterbanks.

   :param y: 1-D mono waveform tensor.
   :param sample_rate: Sample rate in Hz.
   :param n_mels: Number of mel bands. Defaults to 80.
   :param n_fft: FFT size (window length in samples). Defaults to 512.
   :param hop_length: Hop size between frames in samples. Defaults to 160.
   :param f_max: Maximum frequency in Hz. Defaults to 8000.
   :param save_fig: Whether to save figures. Defaults to False.
   :param output_path: Directory to save figures.

   :returns: Tuple of ``(melspec_db, melspec_preemph_db)`` as ``(n_mels, T)``
             numpy arrays in dB.
