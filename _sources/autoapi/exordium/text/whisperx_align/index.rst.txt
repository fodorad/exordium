exordium.text.whisperx_align
============================

.. py:module:: exordium.text.whisperx_align

.. autoapi-nested-parse::

   whisperX wav2vec2 forced alignment (ships with the ``text`` extra).

   Wraps whisperX's per-language wav2vec2 CTC alignment so a **known** transcript
   can be aligned to audio, returning the same ``list[Word]`` as the built-in
   :class:`~exordium.text.alignment.TorchaudioForcedAligner`.  Having both lets you
   cross-check word timings from two independent aligners (useful when validating
   dataset timestamps such as CMU-MOSEI).

   .. note::

      Only whisperX's **alignment** stage is used here.  Its faster-whisper ASR /
      pyannote VAD path relies on ``torchaudio`` APIs removed in the version this
      repo pins, so transcription is done by
      :class:`~exordium.text.whisper.WhisperWrapper` instead (see
      :class:`~exordium.text.alignment.WhisperWordTimestamper`).

   whisperX ships with the ``text`` extra::

       uv sync --extra text

   Forced aligner backed by whisperX's wav2vec2 alignment models.

   Loads (and caches) a language-specific alignment model on first use, then
   aligns the known transcript against the audio.

   :param language: Language code for the alignment model (e.g. ``"en"``).
   :param device_id: Device index. ``-1`` or ``None`` â†’ CPU, ``0+`` â†’ GPU/MPS.

   :raises ImportError: If whisperX (the ``text`` extra) is not installed.

   Example::

       aligner = WhisperxForcedAligner(language="en", device_id=None)
       words = aligner.align("audio.wav", "hey guys what do you want to eat")

      :param language: Optional language code; must match the model's language if

      :returns: Words in chronological order. Empty if *transcript* is blank, or if the

   .. py:method:: align_segments(audio: pathlib.Path | str | numpy.ndarray | torch.Tensor, segments: list[exordium.text.base.Segment], language: str | None = None) -> list[exordium.text.base.Word]

      Align all *segments* in one whisperX call (native batched path).

      whisperX slices the audio per segment internally and returns word times
      already on the full-audio timeline, so long recordings stay bounded
      without the per-segment Python loop of the base implementation.

      Segments too short for whisperX to align are widened first by
      :func:`~exordium.text.base.prepare_segments`.  whisperX pads a short slice up to
      wav2vec2's input floor, but then rescales its timestamps by
      ``duration / (trellis.size(0) - 1)`` â€” a one-row trellis divides by zero and
      aborts the entire recording.  Even when it survives that, a frame-starved
      segment fails backtrack and comes back *untimed*, so its words are silently
      dropped.  Widening the window fixes both.

      Because a widened window may overlap its neighbours, the merged word stream is
      sorted by time before it is returned.

      :param audio: Full audio â€” file path, numpy array, or torch tensor.
      :param segments: Timestamped transcript segments covering the audio.
      :param language: Optional language code (must match the loaded model).

      :returns: Words in chronological order on the full-audio timeline.
